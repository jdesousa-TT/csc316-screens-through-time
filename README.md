# csc316-screens-through-time
Screens Through Time - prototyping and final submission repo

## Project Overview
Screens Through Time is an interactive data story that explores how streaming platforms have reshaped global media visibility over time. Using title data from Netflix, Amazon Prime Video, and Disney+, the project examines changes in format, genre, and international representation across contemporary streaming libraries.

Our goal was to help general viewers better understand how streaming has influenced what kinds of films and television shows become visible, distributed, and consumed in the digital era.

## Contents of Submission

### 1. Code
The website source code is included in this submission. Main project files are organized as follows:

- `index.html` – main project webpage
- `css/` – project styling files
    - `style.css`
    - `cinematic-effects.css`
- `js/` – JavaScript files for the site and visualizations
- `data/` – cleaned datasets used in the project

### 2. Data
The cleaned data used in this project is included in the `data/` folder:

- `amazon_prime_titles.csv`
- `disney_plus_titles.csv`
- `netflix_titles.csv`

These files were used to build the streaming-platform comparisons and interactive visualizations in the final website.

### 3. Libraries and Frameworks
This project uses the following libraries/frameworks:

- D3.js
- Bootstrap 5
- Google Fonts

Any external libraries are linked in the project files where required.

## Visualizations Included
The final website includes several interactive visualizations related to streaming and global media visibility, including:
- genre-based comparisons
- language representation over time
- genre-language relationships
- global streaming map
- country comparisons
- director-based exploration
- Netflix TV season distribution
- revenue vs. budget analysis

## Notes for Running the Project
To run the project locally, open the project folder in a local server environment rather than opening the HTML file directly. This is recommended because the visualizations load CSV data files and may not work correctly through direct file access alone.
